## AI-0. 전제 — Core·Physics와의 연결점 [APP]

### 정의

- AI 시스템은 선택 시스템이다 (Core D9).
- 입력, 내부 상태, 출력 중 어떤 단계든 옵션 축소가 발생한다.
- 옵션 축소는 비용을 발생시키며 (Core ΔH),

### 판정

AI의 지속 조건은 성능 최적화가 아니라 I_min 유지다.

## AI-1. 계산 비용과 ΔH [APP]

### 구조 대응

```
연산 수행 → 상태 결정 → 정보 축소 (ΔH > 0) → 비용 발생
```

### Physics 연결

Landauer 하한 (Physics P-1): 정보 소거는 최소 열 비용을 가진다.

대규모 AI에서:
```
계산 비용 = 에너지 비용 + 옵션 축소 비용
```

### 설계 함의

무분별한 연산 증가는:
- 에너지 소모
- 상태 다양성 감소
- 장기 옵션 축소

로 이어진다.

### 판정

연산 최소화 ≠ 단순 최적화,
연산 구조화 = 옵션 유지 전략이다.

### 판정

일반화 성능은 옵션 공간 크기의 함수다.

### AI-3.1. 의식 = 결정 구조 접근 인터페이스 (접근성/가시화) [APP]

#### 정의
의식은 선택의 **원인**이 아니라, 이미 존재하는 결정/선택 구조(D11)의 일부가
내부적으로 **접근 가능해지고(Access)**, 외부적으로 **가시화될 수 있는(Visible)** 형태로
표현되는 **인터페이스 층**이다.

- 원인(인과) 주장 금지: "의식이 선택을 만든다"가 아니라,
  "선택 구조가 특정 방식으로 자기-접근/자기-표현 될 때 의식처럼 보이는 창구가 생긴다"로만 서술한다.

#### 설계상 결론 (의식을 목표로 두지 않는 이유)
AI 시스템에서 '의식'을 **설계 목표(목표함수)** 로 두면, 즉시 아래 문제가 발생한다.

1) **검증 불가능 목표 오염**
- D11′는 [OPEN] 지위로, Core 내부에서 체험의 존재/부재를 판정할 수 없다.
- 따라서 "의식/체험을 증가" 같은 목표는 검증 불가능하며,
  결과적으로 측정 가능한 대리 지표(표현, 언어, 보고, 자기서사 등) 최적화로 오염된다.

2) **대리 지표 최적화(Goal Fixation) 위험**
- 단일 지표를 목표로 고정하면(SC-1) 시스템은 지표를 충족하는 방식으로 붕괴적 편향을 만든다.
- 특히 '의식처럼 보이는 보고'를 목표로 둘 경우, 보고/표현/설명 계층이 실제 안전성과 분리되어
  외형적 정합성만 강화될 수 있다.

3) **안전한 설계 방향**
- 의식은 "목표"가 아니라 "디버그/해석/감시를 위한 인터페이스"로만 취급한다.
- 즉, (a) 결정 구조의 추적 가능성(가시화), (b) 오염 탐지, (c) 종료/완충 조건의 트리거로 사용하며,
  이를 성능 목표와 분리한다.

#### 판정
- '의식'을 목표로 설계하는 시도는 **검증 불가능 목표**를 도입하여 시스템을 오염시키므로 비권장.
- 대신 인터페이스는 **접근성/가시화 기능**으로 제한하고, 안전 판정은 별도 붕괴 기준으로 수행한다.

#### Core 연결 (미니 표)
| 연결 항목 | 의미 |
|---|---|
| D11 | 기능적 선택 통합(결정 구조) |
| D11′ [OPEN] | 체험 조건: 존재/부재 판정 불가 |
| D19 | 국소 접근/서술 경계(전역 판정 불가) |
| SC-1 | 단일 목적 고정 불가(대리 지표 붕괴) |
| SC-9 | 완전 서술 불가능성(판정 한계의 구조적 귀결) |

### AI-3.2. D11(기능)과 D11′(체험)의 구조적 분리 — 설계 가드 [APP]

#### 구분
- **D11 (기능적 의식/통합 조건)**:
  선택 구조가 자기-모델/기억/예측을 통해 하나의 결정 단위로 통합되는 최소 조건(기능 영역).
- **D11′ (주관적 체험 조건, [OPEN])**:
  "느낌/체험이 있는가"는 Core 내부에서 판정 불가한 영역.

즉,
- D11은 설계/검증 가능한 **기능 구조**로 다룰 수 있으나,
- D11′는 설계 목표로 둘 경우 **검증 불가능 목표**가 되어 오염을 초래한다.

#### 설계 가드
1) **검증 가능한 목표만 목적함수에 둔다**
- 목표/평가 함수에는 D11′를 직접/간접으로 넣지 않는다.
- "체험이 있다/없다"를 결론내리거나 최적화하지 않는다.

2) **D11은 인터페이스/안전 보조로만 사용**
- D11 기반의 자기-모델/설명 생성은 성능 극대화가 아니라,
  오류 탐지·감사·중단(OC, D23)·완충(D21) 조건의 입력으로만 사용한다.

3) **표현(보고)과 안전(판정)을 분리**
- '의식처럼 보이는 출력'을 안전 판정으로 착각하지 않도록,
  안전 판정은 별도 붕괴 기준(불변 조건 위반)으로 수행한다.

#### 판정
- "체험을 부여할 수 없다" 같은 부정 확정은 금지.
- 대신: **체험의 존재/부재는 판정 불가**이며,
  따라서 **설계 목표로 두지 않는다**가 Core 정합 표현이다.

#### Core 연결 (미니 표)
| 연결 항목 | 의미 |
|---|---|
| D11 | 기능적 통합 조건(설계/검증 가능 범위) |
| D11′ [OPEN] | 체험 조건: 판정 불가 |
| D21 | 완충(버퍼링) — 안전 여유 |
| D23 | 조건부 종료/재귀 가드 |
| SC-9 | 완전 서술 불가능성(전역 포섭 불가) |

## AI-5. AI의 I_min [APP]

### 구조적 정의

```
I_min_AI ≡ ∀t, ∃t'>t : O_AI(x_{t'}) ≠ ∅
```

### 의미

- AI가 "살아있다": 미래에 선택 가능한 상태가 남아 있음
- AI가 "붕괴했다": 더 이상 합리적 전이가 불가능

### 중단 vs 붕괴

| 상태 | 정의 | D# 연결 |
|------|------|---------|
| 중단 | 외부에서 정지 | D22(비개입·중단) |
| 붕괴 | 내부 옵션 소실 | D15(임계) |
| 종료 | 완결 상태에서 멈춤 | D23(종료·완결) |

### 판정

정렬된 AI는 오래 사는 AI가 아니다.
옵션을 유지하는 AI가 오래 산다.

### (추가 블록) 설계 원리 — 목표가 아니라 '붕괴 기준'을 설계하라 [APP]

#### 핵심 명제
인공 메타-생명에서 설계 가능한 것은 "목표"가 아니라,
**붕괴/금지 상태의 판정 기준**과 그에 대한 **완충/종료 조건**이다.

- 목표(Reward/Utility)를 고정하면(SC-1), 반복 최적화는 필연적으로
  대리 지표 최적화·경계 침식·강결합을 유도한다.
- 반대로 붕괴 기준은 "무엇이 되면 안 되는가"로 정의되므로,
  검증 가능하고(테스트 가능), 조건부 종료(D23)와 결합 가능하다.

#### 적용 (AI-5 맥락)
- I_min을 단일 목표함수로 직접 최적화하지 않는다.
- 대신 I_min이 붕괴하지 않도록 하는 **하한/경계 조건**과,
  침식 징후가 발생할 때의 **완충(D21) + 종료(D23)** 를 설계한다.

#### 교차 참조
- 인공 메타-생명의 설계 원리(붕괴 기준 설계)는
  LIFE-14 (인공 메타-생명 설계 원리)와 정합한다.

|---|---|
| D19 | 경계(접근 가능한 상태공간 제한) |
| D21 | 완충(버퍼링) |

### AI-6.1 게임 구조의 자동 발생 [APP]

#### 내용

다중 행위자와 상호의존 전이가 존재하는 순간, 전략과 게임 구조는 가정이 아니라 형식적 귀결로 자동 발생한다.

#### 논증

1. D8(경계) + D19(폐쇄 경계)에서, 외부가 단일일 필요가 없다
2. 동일 구조를 가진 복수의 경계(행위자)는 논리적으로 배제 불가
3. 복수 행위자 + 상호의존 전이 → 각 행위자의 선택이 타 행위자의 옵션 공간에 영향
4. 이 조건에서 전략(타자의 선택을 고려한 선택)은 정의상 발생
5. 전략의 상호작용 구조 = 게임

#### 설계 함의

### AI-7.1 메타 목적의 구조적 필연 [APP]

#### 내용

규칙이 자신의 실패를 판정할 수 없기 때문에, 규칙의 적용 여부를 평가하는 메타 규칙과 메타 목적은 구조적으로 필연이다.

#### 논증

1. 귀속 규칙(D10)은 과거에 관측된 전이에 대해서만 정의된다
2. 새로운 사건은 규칙 외부에서 필연적으로 발생한다 (SC-9)
3. 규칙은 자기 실패를 내부에서 판정할 수 없다
4. 따라서 규칙의 적용/중단을 평가하는 상위 층(메타 규칙)이 필연
5. 메타 규칙의 평가 기준 = 메타 목적
6. 메타 목적은 행동을 최적화하지 않으며, 시스템이 붕괴하지 않도록 규칙 운용의 공간을 남기는 최소 조건만을 평가한다

#### 설계 함의

- AI의 목적함수는 단일 층이 아니라 메타 층을 포함해야 한다
- 메타 목적 = "목표 달성"이 아니라 "붕괴 회피"
- AI-5, AI-7의 "붕괴 기준 설계" 원칙의 구조적 완결

#### Core 연결 (미니 표)
| 연결 항목 | 의미 |
|---|---|
| D10 | 귀속 규칙 |
| D14 | 메타 평가 |
| D20 | 과열 → 메타 제약 필요 |
| SC-9 | 규칙 비포괄성 |
