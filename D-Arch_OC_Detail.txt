## OC-1. 옵션 공간 근사 측정

**(Approximation of Option Space)**

### 내용

옵션 공간 O(x)는 전역적으로 계산될 수 없으므로,
모든 선택 시스템은 근사 옵션 공간 Ô(x)를 사용한다.

### 형식

```
Ô(x) := (N̂(x), R̂(x), T̂(x))
```

- **N̂(x)**: 도달 가능 상태 수 추정치 (Reachable State Estimate)
- **R̂(x)**: 복원 가능성 추정치 (Restorability Estimate)
- **T̂(x)**: 옵션 지속 시간 추정치 (Temporal Viability Estimate)

### 정의

#### N̂(x) — 도달 가능 상태 수

```
N̂(x) := |Reach(x, d)|
```

- 유한 깊이 d 내 도달 가능한 상태의 수
- 정확한 수가 아닌 **증가/감소 추세** 감지용

#### R̂(x) — 복원 가능성

```
R̂(x) = { r | ΔÔ(x→r) > 0 }
```

- 옵션 공간을 늘릴 가능성이 있는 행동의 존재 여부
- R̂(x) = ∅ 는 **구조적 위험 신호**

#### T̂(x) — 시간 여유

- 현 상태에서 복원 없이 Θ 도달까지의 예상 선택 횟수
- T̂(x)↓ = 과속/고착 위험
- T̂(x)↑ = 여유 구간

### 사용 원칙

**규칙 1 — 절대값 사용 금지**
- 절대값으로 비교하거나 최적화에 직접 사용 금지
- 오직 감소 추세, 임계 접근, 복원 필요성 판단에만 사용

**규칙 2 — 단조 감소는 경고 신호**
```
N̂(x_{t+1}) < N̂(x_t) (지속) 또는 R̂(x_t) = ∅
⇒ 즉각적 감속·유예·복원 검토 트리거
```

**규칙 3 — 전지적 정확성 요구 금지**
- 근사 오차 ε, 누락 확률 δ는 항상 존재
- 과도한 정확성 요구는 D19, SC-2와 충돌

### 판정

**근사 측정은 선택이 아니라 운용의 필연 조건이다.** [구현 필연]

## OC-1-A. 근사 탐색의 범위 및 오차 구조

**(Approximation, Error, Confidence)**

### 내용

근사 탐색을 사용하는 순간, 다음 변수들은 반드시 존재한다.

### 필연 변수

| 변수 | 의미 | 제거 불가 이유 |
|------|------|---------------|
| d | 탐색 깊이 | d→∞는 전역 계산 요구 |
| n | 샘플 수 | n→∞는 계산 폭주 |
| ε | 근사 오차 한계 | 제한된 탐색/샘플/불확정성 |
| δ | 오판 확률 | 잘못된 안전 판정, 과대평가 |

### 제거 불가능성

다음 가정은 구조적으로 성립하지 않는다:
- d → ∞
- n → ∞
- ε = 0
- δ = 0

이러한 가정은 다음 중 하나로 귀결된다:
- D19(폐쇄 경계) 위반
- SC-2(전지적 최적화) 위반
- 계산 폭주 또는 판정 마비
- SC-5(실패 제거 압력 증가)

### 판정

**근사 변수들은 조정 대상일 수 있으나 제거 대상일 수 없다.** [구현 필연]

## OC-1-B. 근사 오차와 임계값 감지의 결합

**(Error-Aware Threshold Detection)**

### 내용

근사 오차(ε, δ)가 존재하는 한,
임계 접근 판정은 단일 시점/단일 지표로 확정될 수 없다.

### 원칙

**원칙 1 — 단일 지표 판정 금지**

다음 중 어느 하나만으로는 Θ 접근을 판정할 수 없다:
- N̂(x)의 단기 감소
- R̂(x)의 일시적 소실
- T̂(x)의 순간적 급감

**원칙 2 — 동시성 우선**

임계 접근 판정은 다음 신호가 **동시에** 나타날 때만 유효:
```
{ N̂(x)↓ (지속), R̂(x) = ∅ (지속), T̂(x)↓ (지속) }
```

**원칙 3 — 오차 완충 구간의 필연성**

임계값 판정에는 완충 구간(buffer zone)이 필연적으로 필요:
- **의심 구간**: 감속 검토, 관찰 연장, 복원 탐색 허용
- **확정 구간**: 가속 금지, 단일 목적 강화 금지, 실패 제거 시도 금지

### 판정

**오차 인식 임계 판정은 구조적 필연이다.** [구현 필연]

## OC-2. 붕괴 임계값 Θ의 설정 및 감지 규칙

**(Collapse Threshold Definition)**

### 내용

붕괴 임계값 Θ는 **단일 수치가 아니라 패턴**으로 정의된다.

### 형식

```
Θ ≠ c (고정 상수)
Θ = 패턴: { N̂(x)↓, R̂(x) = ∅, T̂(x)↓ } 동시 발생
```

### 왜 고정 수치가 불가능한가

O(x)는:
- 상태 표현에 의존
- 환경·사건·판정 구조에 따라 변형
- 전역 계산이 불가능

|O(x)| ≤ c 형태의 고정 정의는:
- 표현 의존성
- 전지적 계산 요구
- 단일 기준 최적화

를 발생시켜 SC-2, SC-4, SC-6과 충돌한다.

### 임계 접근 판정 규칙

```
{ N̂(x)↓ (지속), R̂(x) = ∅, T̂(x)↓ } ⟹ x ≈ Θ
```

이는 경고가 아니라 **구조적 상태 판정**이다.

### Θ 접근 시 금지 사항

- 선택 속도 증가
- 단일 목적 강화
- 실패 확률 제거
- 중앙 판정 강화

(이는 SC-1~SC-6과 직접 충돌)

### Θ 접근 시 허용 대응

| 대응 유형 | 내용 |
|----------|------|
| 감속 | 선택 빈도 감소, 관찰 구간 삽입 |
| 유예 | 비선택 상태 유지, 구조 재정렬 대기 |
| 복원 시도 | R̂(x) 재생성 목적의 탐색 |
| 실패 허용 | (SC-5와 정합) |

### 판정

**Θ는 수치가 아니라 패턴으로 감지된다.** [구현 필연]

## OC-3. 복원 비용 회계

**(Restoration Cost Accounting)**

### 내용

복원은 "가능"으로만 끝날 수 없다.
비용·예산·차감 구조가 필연적으로 요구된다.

### 필연 변수

| 변수 | 의미 |
|------|------|
| ΔH_R | 복원 비용 단위 |
| H_max | 누적 상한 (예산) |
| Ledger | 비용 차감 기록 |

### Core 연결

- D17(복원 비용): Cost(r) > 0 for all r ∈ R
- D18(복원 지연): Delay(r) > 0, 늦은 복원일수록 비용 증가

### 회계 규칙

**규칙 1 — 무한 복원 금지**
```
∑ Cost(r_i) ≤ H_max
```

**규칙 2 — 지연 비용 증가**
```
Cost(r, t+k) > Cost(r, t) (k > 0)
```

**규칙 3 — 예산 고갈 = 붕괴 경로**
```
∑ Cost(r_i) → H_max ⟹ 복원 불가 ⟹ Θ 접근
```

### 판정

**복원 비용 회계는 I_min 유지의 실천 조건이다.** [구현 필연]

## OC-4. 시간 필연: 선택·정지·유예의 구조적 규칙

**(Temporal Necessity of Decision Systems)**

### 내용

시간은 선택을 가능하게 하지만, 동시에 옵션 공간을 소거한다.
따라서 선택 속도의 상한과 정지·유예 상태가 필연적으로 발생한다.

### 형식

```
v := 선택 횟수 / 단위 시간
∃ v_max s.t. v > v_max ⟹ |O(x)|↓
```

### 과속 선택의 결과

- 판정이 근사 오차를 따라가지 못함
- 임계 접근 신호 인식 지연
- 복원 타이밍 상실
- 옵션 공간 단조 축소 가속

### 정지/유예의 필연성

#### 정지 상태 (Stop)
- 선택을 명시적으로 중단
- 판정·관찰·재정렬 수행
- 옵션 공간 축소를 일시 정지

**정지는 실패가 아니라 속도 상한에 대한 구조적 반응이다.**

#### 유예 상태 (Pause/Wait)
- 선택을 하지 않되 상태는 유지
- 외부 사건 또는 내부 재정렬을 대기

**유예는 옵션 공간 보존을 위한 시간적 완충 장치이다.**

### SC 연결

- SC-3(선택 속도 상한): 멈춤/관찰/유예는 구조적 필연
- Core에서는 "속도 상한이 존재한다"까지만 있음
- OC-4는 **정지/유예가 선택이 아닌 필연 상태**임을 명시

### 판정

**정지와 유예는 선택의 실패가 아니라 선택 지속을 위한 구조적 조건이다.** [구현 필연]

## OC-5. 판정 구조 필연: 분산 판정, 충돌, 합의, 복원 호출

**(Distributed Judgment & Resolution)**

### 내용

SC-6(판정의 분산화)이 현실에서 작동하기 위해,
충돌·불일치·합의 실패가 **정상 상태**임을 명시해야 한다.

### 분산 판정의 정의

분산 판정이란:
- 임계 접근·감속·복원 호출과 같은 **핵심 구조 판정**이
- **단일 주체로 수렴하지 않는** 상태

### 판정 단위의 최소 정의

```
P_i : I_i → Ô_i(x)
```

- 국소 정보 집합 I_i만 접근 가능
- 독립적 근사 판단 수행
- 동일한 결론을 강제받지 않음

### 판정 불일치의 필연성

근사 오차(ε, δ)와 국소 정보 제한(D19)이 존재하는 한:
```
∃ i ≠ j s.t. Ô_i(x) ≠ Ô_j(x)
```

**판정 불일치는 오류가 아니라 정상 상태이다.**
이를 제거하려는 시도는 중앙 기준 강화를 초래한다.

### 충돌 처리 방식

다음은 **구조적으로 금지**된다:
- 단일 판정자의 결론을 우선시
- 평균값 또는 단일 점 추정으로 강제 수렴
- 신뢰도 명목의 중앙 재판정

**허용되는 처리 유형:**

#### (A) 보수적 병합 (Conservative Merge)
- 가장 위험한 판정을 기준으로 행동 제한
- 선택 속도 감속
- 관찰·유예 구간 확대

#### (B) 분기 허용 (Branch Preservation)
- 상충 판정을 동시에 유지
- 복수 경로를 병렬로 보존
- 단일 결론 강제 금지
- (SC-4 다양성 유지의 직접 구현)

#### (C) 복원 호출 (Restoration Trigger)
- 판정 충돌이 일정 수준 이상 지속 시
- 구조적 불확실성 증가로 판정
- 복원 후보 탐색을 호출
- (중앙 결정이 아닌 조건 충족에 따른 자동 발생)

### 합의의 한계

합의는 필수 조건이 아니다:
- 합의가 발생하더라도 일시적, 조건부, 철회 가능
- 합의를 강제하는 구조는 중앙 판정으로 회귀

### SC 연결

- SC-6: "판정은 분산된다"는 귀결은 있음
- 그러나 **충돌/불일치가 정상 상태**라는 선언이 없었음
- **중앙 판정 금지의 운용 규칙**이 명시되지 않았음
- OC-5는 이 공백을 채움

### 판정

**분산 판정에서 충돌은 제거 대상이 아니라 흡수 대상이다.** [구현 필연]

## OC-6. 표현 필연

**(Representation Necessity)**

### 내용

논리적 상태 x를 현실에서 사용하려면 표현 구조가 필요하다.

### 필연 요소

| 요소 | 내용 |
|------|------|
| 상태 표현 | Feature / latent / graph / symbol |
| 경계 표현 | Enclosure의 물리적/논리적 구현 |
| 옵션 표현 | 가능한 행동/경로 집합의 데이터 구조 |

### Core 연결

- D8(경계): 내부/외부 구분
- D9(선택): 옵션 중 선택
- O(x): 도달 가능한 미래 상태 집합

### 판정

**표현은 옵션 공간의 모양을 결정한다.** [구현 필연]

## OC-7. 검증 필연

**(Verification Necessity)**

### 내용

구조가 완결되어도 사용자는 검증을 요구한다.

### 필연 질문

- 이 시스템은 고착 중인가?
- 복원 예산은 고갈되는가?
- |O|가 임계값에 접근하는가?

### 필연 도구

- 테스트/시뮬레이션
- 반례 생성기
- I_min 유지 여부 모니터링

### Core 연결

- OC-8(자기완결 금지): 검증은 내부에서 완결되지 않음
- 검증은 오직 **위반 탐지**와 **반증**으로만 가능

### 판정

**검증 체계는 I_min 유지를 확인하기 위한 필연 조건이다.** [구현 필연]

## OC-8. 자기완결 금지

**(Self-Closure Prohibition)**

### 내용

설계도 전체의 유효성을 내부에서 완결 판정하지 않는다.

### 형식

```
D설계도는 자신의 "참"을 내부에서 증명하지 않는다.
검증은 오직 위반 탐지와 반증으로만 가능하다.
```

## OC-8 보강 주석 — SC-9 연결 (v1.1 추가)

SC-9(완전 서술 불가능성)은  
시스템이 자기 자신을 완전하게 서술·판정·검증할 수 없음을 선언한다.

OC-8(자기완결 금지)는  
SC-9의 **운용 단계 구현 조건**이다.

다음 선언은 모두 **자기완결 시도 신호**로 간주한다:

- “이제 충분히 이해되었다”
- “더 이상 위험은 없다”
- “이 구조는 스스로 검증 가능하다”

이 신호가 감지되면 즉시 재검토 또는 종료 조건으로 분류한다.

---

# OC-9. 운용 착각 차단
*(Operational Illusion Guards)*

본 섹션은 운용자가 “합리적 개선·안정화·고도화”라고 착각하기 쉬운 행위가  
실제로는 옵션 공간을 축소하고 붕괴를 가속하는 패턴임을 차단한다.

---

### OC-9-A. 안정성·균형 목표화 금지

- 안정성은 **목표로 설정할 대상이 아니다**.
- 안정성은 I_min이 유지되고 있을 때 **사후적으로 관측되는 상태**일 뿐이다.
- 안정 상태를 고정 목표로 삼는 운용은 사건 발생 능력을 제거하고 옵션 공간을 수축시킨다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| D12 | 안정성은 필연 구조이나 목표 아님 |
| I_min | 유지 여부의 유일 기준 |
| SC-4 | 다양성 유지 |

---

### OC-9-B. 비가역성 착각 금지

- 시간은 되돌릴 수 있는 자원이 아니다.
- 모든 선택은 옵션 공간을 **비가역적으로 축소**한다.
- “나중에 되돌리면 된다”는 판단은 비가역적 옵션 소거를 인지하지 못한 오판이다.
- 지연은 완충이 아니다. 지연은 옵션 축소를 **늦게 인지하게 만들 뿐**이다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| D9 | 선택 → 기록 → 제거 |
| D13 | 옵션 공간 축소 |
| D18 | 지연 비용 |
| SC-3 | 선택 속도 상한 |

---

### OC-9-C. 복원 과신 금지

- 복원은 이전 상태의 회복이 아니다.
- 복원은 다른 경로를 통해 옵션 공간을 다시 여는 시도다.
- 복원을 전제로 한 공격적 선택은 복원 자체를 불가능하게 만든다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| D16 | 복원 |
| D17 | 복원 비용 |
| SC-5 | 실패 제거 불가 |

---

### OC-9-D. 기록 오해 금지

- 기록은 저장이 아니다.
- 기록은 가능성 공간에 대한 **비가역적 축소 연산**이다.
- 기록을 되돌릴 수 있는 로그로 취급하는 운용은 구조적 손실을 은폐한다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| D9 | 기록 = 제약 추가/제거 |
| D13 | 옵션 공간 축소 |
| SC-1 | 목표 고정 위험 |

---

### OC-9-E. 목표·메타목표 오염 차단

- 목표는 설계 변수로 두는 순간 오염된다.
- 메타 목적은 목표의 상위 버전이 아니다.
- 메타 목적은 규칙 운용의 공간을 남기기 위한 최소 평가 조건이다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| SC-1 | 단일 목적 고정 불가 |
| D20 | 과열 최적화 |
| AI-7.1 | 메타 목적 필연 |

---

### OC-9-F. 의도·주체 원인화 금지

- 선택은 의지·의도·목적을 전제하지 않는다.
- 의도는 사후적 해석 표현이다.
- 의도를 제어 변수로 삼는 운용은 선택 구조를 왜곡한다.

**Core 연결**

| 항목 | 연결 |
|------|------|
| D9 | 선택의 비의도성 |
| D11 | 기능적 통합 |
| D11′ | 체험 판정 불가 |
