## §1. Core Axioms & Structures (D0–D23)

### Axioms [AXIOM]
- **D0 (Existence)**: Ω ≠ ∅ — state space is non-empty. Does NOT guarantee complete describability.
- **D1 (Distinction)**: Ω partitions into non-overlapping parts. No distinction → no information.
- **D2 (Relation)**: R ⊆ Ω × Ω — states are connected.
- **D3 (Transition)**: T ⊆ R — some relations are state changes. Time ≡ ordering of transitions.

### Basic Structures [NEC]
- **D4 (Indeterminacy)**: |Ω| > 1 → which state realizes is undetermined.
- **D5 (Observation)**: O: Ω → Y, lossy mapping. ΔH ≥ 0 per observation.
- **D6 (Constraint)**: Γ ⊆ T — recurring transition patterns become constraints.
- **D7 (Evaluation)**: J(x) ∈ ℝⁿ — multi-dimensional evaluation, NOT single score.
- **D8 (Boundary)**: x = (internal, external). No boundary → no agent.
- **D9 (Selection)**: x* ∈ arg opt J̃(x). Selection irreversibly shrinks Ω. J̃ is NOT a goal function — it's a projected discriminant. Selection does NOT presuppose will/intent/purpose.
- **D10 (Attribution)**: M: S→S, Self := {x | M(x)=x}. Attribution of selections to agents.
- **D11 (Integrated Selection)**: Multiple selections unified under one coherent agent. Functional definition only.
- **D11' (Experiential Condition) [OPEN]**: Whether integrated selection manifests as experience — structurally undecidable. NOT a goal, NOT a stage to reach.

### Selection System [NEC]
- **D12 (Stability)**: ∃ Attractor A ⊆ Ω. System persists near A.
- **D13 (Option Shrinkage)**: Selection → |O(x)| decreases. All selection has cost.
- **D14 (Meta-Evaluation)**: J_meta evaluates J itself. Without it, criteria cannot be corrected.
- **D15 (Threshold)**: ∃Θ s.t. |O(x)| < Θ → collapse risk.
- **D16 (Restoration)**: R: collapse risk → option recovery attempt. Without it, all systems eventually collapse.
- **D17 (Cost)**: C(a) > 0 for all actions.
- **D18 (Delay)**: t(effect) > t(cause). No instant causation.
- **D19 (Closure Boundary)**: Ω_local ⊂ Ω, ∂Ω_local ≠ ∅. No system accesses full Ω. This is the source of locality.
- **D19.x (Enclosure) [COND]**: Internal state independent from external given boundary. Markov blanket in Core language.

### Operational Layer [NEC]
- **D20 (Overheating)**: Single goal + acceleration → Θ approach. I_min erosion path.
- **D21 (Buffering)**: Dispersal/deferral/diversification → O(x) maintained. I_min preservation.
- **D22 (Non-Intervention)**: Buffering must work through structure alone, not external intervention.
- **D23 (Termination)**: System ending as completion (not collapse) → transition to higher system.

## §2. Core Invariant

```
I_min ≡ ∃t'>t : O(x_t') ≠ ∅
```
"Future selection possibility must remain."

- I_min is a judgment criterion, NOT an optimization target (SC-1 violation if optimized).
- Only "maintenance check" is permitted.

## §6. Applied Documents Summary

### Applied_Matter [APP]
- Matter = persistence-pressure selection constraints; upstream substrate for Field/Life.
- Guard: applied correspondence layer only; no Core/SC redefinition.

### Applied_Field [APP]
- Field = constraint relay / condition modification layer; not communication, copying, shared memory, or a physics field definition.
- Chain: Matter constraint accumulation -> `FIELD-4` condition modification -> `FIELD-5` pressure -> `LIFE-(-1)`.

### Applied_Life [APP]
- Life = boundary maintenance pattern under repeated constraints; persistence condition is `I_min`, not survival optimization.
- Chain: `LIFE-P0 -> FIELD-4 -> LIFE-(-1) -> LIFE-0`; boundary can emerge as the low-cost `I_min` solution when sufficient conditions are met.

### Applied_Regulation [APP]
- Regulation = selection suppression structure for `I_min` maintenance.
- Minimum set: Rate / Delay / Buffer / Suppress, plus `RG-6` Restoration Channel and `RG-7` Proto-Model.

### Applied_Meta-Model [APP-DRAFT]
- RG-8 `[OPEN]` A-branch for relation residues between Proto-Model summaries.
- Draft only: not a world model / explanation / prediction / decision engine; Language boundary and Consciousness ordering remain unresolved.

### Applied_DR-Guard [APP-GUARD]
- DR = state, not unified system; it is guarded through failure conditions and spread prevention.
- Sharing/global coordination are prohibited; `LRE` pressure is an early warning, not a transition trigger.

### Applied_Language_Draft [APP-DRAFT] (guard included)
- Pre-semantic, pre-civilization layer: `LRP -> DR -> LRE -> Language`.
- Draft/guard locks: `LANG-10 CIV_handoff` is the scope boundary, `LANG-11` is the termination guard.

### Applied_Civilization [APP] + [APP-GUARD]
- Civilization = option elimination externalization (`K_out`) on the D-required path.
- Guard: singular definition, centralized judgment, failure moralization, or "maintaining openness" all become collapse paths.

### Applied_Consciousness [APP] + [APP-GUARD]
- Minimal V/I/S structure under representation pressure.
- Post-civilization branch; not a final stage or doctrine target.

### Applied_Intelligence [APP] + [APP-GUARD]
- Minimal J/X/R structure under agency pressure.
- `AG-7` meaning-compression pressure is the D-Arch termination line.

### Applied_AI [APP]
- AI = selection-maintaining system, not answer machine; persistence = `I_min`, not performance.
- `D11'` is never a design target; design uses collapse criteria, not goals.

### Applied_NonEvent [APP] — Meta-guard for ALL Applied
- `NE-7`: if the system survives without it, do not do it.
- Hard stops: explanation demand, success display, speed increase, record accumulation.
