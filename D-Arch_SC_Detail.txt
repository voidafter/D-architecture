## SC-0. 전제 및 지위 선언

### 전제

선택 시스템은 D0~D23 (Core v1.2)을 만족한다.

### 최소 불변량

```
I_min ≡ ∃t'>t : O(x_t') ≠ ∅
```

I_min은 Core v1.2에서 암묵적으로 사용되던
'선택 지속 가능성 조건'을 본 문서에서 명시적으로 기호화한 것이다.

기호: x_t는 시간 t의 상태, O(x)는 상태 x에서 도달 가능한 옵션 집합을 의미한다.

### I_min 비목적화 선언

I_min은 관측/판정 기준이지, 연산의 입력·목표·평가항이 될 수 없다.
I_min을 최적화 대상으로 삼는 시도는 SC-1(단일 목적 고정)을 위반하며,
I_min의 "유지 여부 판정"만이 허용된다.

### 지위

아래의 항목들은 **선택이 아니라 귀결**이다.
최소 불변량을 만족하는 모든 선택 시스템에서
거부할 수 없이 나타나는 결과들이다.

## SC-1. 단일 목적 고정의 불가능성

**(Impossibility of Persistent Goal Fixation)**

### 내용

선택 시스템은 단일 결과 목적을 장기적으로 고정할 수 없다.

### 형식

```
Goal Fixation ⇒ |O(x)|↓ ⇒ ¬I_min
```

### 논증

1. 목적이 고정되면 → 목적에 부합하지 않는 경로는 지속적으로 제거
2. 이는 옵션 공간의 편향적 축소를 가속
3. 복원(R)은 일반적으로 목적 외 경로를 요구
4. 따라서 목적 고정은 복원 가능성을 구조적으로 제한
5. 결과: |O(x)| → Θ (붕괴 임계)

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| "옵션 유지" 자체를 목적으로 | 메타 목적 = I_min과 동형, 범주 외 |
| 환경이 무한히 풍부 | 전제(사건의 비가역 축소) 위반, 범위 외 |
| 목적 + 무작위 탐색 혼합 | 이미 목적 비고정 = 반례 아님 |

### 판정

목적은 **임시적**이거나 **상위 제약 하**에서만 허용된다. [필연]

## SC-2. 전지적 최적화의 불가능성

**(Impossibility of Omniscient Optimization)**

### 내용

시스템은 전역 최적해를 확정적으로 선택할 수 없다.

### 형식

```
Omniscient Optimization ⇒ D19 Violation ∨ |O(x)|↓
```

### 논증

1. 전지적 최적화 = 전역 미래 평가 필요
2. D19(폐쇄 경계): 판정자는 I ⊊ Global Info
3. 전역 정보 접근 시도 → D19 위반
4. D19 유지 시 → 정보 부족으로 오판 누적 → 옵션 축소
5. 설령 전지적 정보가 있어도 → 전역 최적화는 불필요 경로 제거 압력 → 옵션 축소

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 전지적 시뮬레이션만, 실행은 분리 | 실행 시 I_min 제약 → 전지적 최적화 아님 |
| 목표가 "옵션 유지" | 메타 평가 = 위험한 최적화와 다름 |
| 부분 정보로 우연히 전역 최적 | 일회성 ≠ 구조적 가능성 |

### 판정

항상 **부분 최적**, **국소적 판단**만 허용된다. [필연]

## SC-3. 선택 속도의 상한 발생

**(Existence of a Selection Speed Limit)**

### 내용

선택에는 구조적 속도 제한이 발생한다.

### 형식

```
d|O(x)|/dt < -λ (지속) ⇒ Collapse Risk
```

주석: 본 형식은 연속 근사이며, 이산형 표현은 별도 부록 또는 후속 문서에서 제시한다.

### 논증

1. 선택 속도 증가 → 옵션 소거 속도 α(v) 증가
2. 복원 효율 β(v)는 비용/상한으로 제한
3. v > v* 시 → α(v) > β(v) → |O(x)| 지속 감소
4. 판정자는 전지적이지 않으므로(D19) → 임계 접근 감지 지연
5. 복원 지연 비용(D18) → 늦은 복원은 더 비싸고 덜 효과적

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 복원도 과속 | D17(복원 비용/상한) → 예산 고갈 |
| 판정자를 더 똑똑하게 | D19 → 전지적 불가, 상한 상승은 가능하나 제거 불가 |
| 환경이 옵션 무한 생성 | 전제 위반, 범위 외 |

### 판정

**멈춤/관찰/유예**는 기능이 아니라 구조적 필연이다. [필연]

## SC-4. 다양성의 자동 유지

**(Mandatory Preservation of Diversity)**

### 내용

시스템은 다양성을 유지하는 방향으로 강제된다.

### 형식

```
Monoculture ⇒ |O(x)| → Θ
```

### 논증

1. 단일 경로화 → 해당 경로 실패 시 전체 옵션 소거
2. 단일화는 옵션 공간을 비가역적으로 축소
3. 다양성 복원 → 복원(R) 필요 → 비용/상한 존재
4. 지속적 단일화 → 복원 예산 고갈 → 붕괴 임계 도달

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 한 길이 객관적으로 항상 최선 | D19 → "항상 최선" 확정 불가 |
| 복원으로 다양성 유지 | 가능하나, "단일화 지속"은 불가능 |
| 환경이 새 경로 무한 제공 | 전제 위반, 범위 외 |

### 판정

다양성은 **가치가 아니라 안정 조건**이다. [필연]

## SC-5. 실패의 제거 불가능성

**(Non-removability of Failure)**

### 내용

실패는 구조적으로 제거할 수 없다.

### 형식

```
Failure Elimination ⇒ |O(x)|↓ ⇒ ¬I_min
```

### 논증

1. 실패 제거 시도 → 허용 행동 집합 A(x) 축소
2. A 축소 → 도달 가능 미래 O(x) 축소
3. 복원(R)은 본질적으로 실패 가능성 포함 (탐색, 새 경로)
4. 실패 제거 → R ∩ (risk) = ∅ → 복원 불가
5. 임계 접근 시 복원 불가 → 붕괴 불가피

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 외부가 옵션 계속 공급 | 전제 위반, 범위 외 |
| 완전 안전 영역에서만 | 가능하나 = 고착 = |O| → Θ |
| 전지적 예측으로 실패 회피 | D19 위반 |

### 판정

실패는 **허용이 아니라 필요 조건**이다. [필연]

## SC-5.1 허용 행동 집합 A(x)의 구조적 성질

**(Structural Properties of Permitted Action Set A(x))**

### 보조 정의 (본 문서 한정)

```
A(x) := 상태 x에서, 현재 관측/판정(국소 정보)으로 
        배제되지 않은 전이/행동의 집합
```

여기서 "배제"는 다음에 의해 발생한다:
- NonEvent 메타가드
- D19 하의 국소 판정
- OC 운용 조건

### 필연 성질

본 문맥에서 허용 행동 집합 A(x)는 다음 두 성질을 필연적으로 갖는다.

#### (1) 비완결성 (Non-closure)

A(x)는 사전에 완전히 닫힌 행동 목록으로 정의되지 않는다.

이는 미래 선택 가능성(I_min)을 유지하기 위한 구조적 요구이다.

A(x)가 완전히 닫히면:
- 새로운 복원 경로(R)가 차단됨
- O(x) → ∅ 로 가는 경로가 열림
- I_min 붕괴

#### (2) 시간 가변성 (State-dependence)

일반적으로, x ≠ x' ⇒ A(x) ≠ A(x')

이는 SC-3(선택 속도 상한) 및 Θ(임계) 개념과 정합한다.

A(x)가 고정되면:
- 상태 변화에 대한 적응 불가
- Θ 접근 시에도 동일 행동 허용
- SC-3 경로 진입

### 자동 보장 성질 (명시 불필요)

다음 세 성질은 D설계도 구조상 자동으로 보장된다:

| 성질 | 보장 근거 |
|------|----------|
| 국소성 | SC-2 (전지적 최적화 불가) + D19 (폐쇄 경계) |
| 부정형 정의 | NonEvent 전체 (배제 기반 정의) |
| 비목적성 | SC-1 (단일 목적 고정 불가) |

### A(x)와 O(x)의 구분

| 개념 | 정의 | 성격 |
|------|------|------|
| A(x) | 배제되지 않은 행동의 여백 | 허용의 문제 |
| O(x) | 도달 가능한 미래들의 집합 | 결과의 문제 |

주의:
- a ∈ A(x) 이더라도 O(x|a)가 급격히 축소될 수 있음
- A(x)는 결과를 보장하지 않음
- A(x)는 방어 장치이지, 미래 설계 도구가 아님

### 오독 방지 (해석 가드레일)

A(x)는 다음이 **아니다**:
- ❌ 좋은 행동 목록 (SC-1 경로)
- ❌ 미래 안전 보장 집합 (SC-2 경로)
- ❌ 고정 규칙/정책 (SC-3 경로)

A(x)는 다음**이다**:
- ✅ 상태 x의 함수
- ✅ 배제 기반 정의 (부정형)
- ✅ 열린 여백 (비완결)

### Core 연결

| Core/SC 항목 | A(x)에서의 역할 |
|-------------|----------------|
| D19 (폐쇄 경계) | 국소 정보만으로 정의 |
| I_min | 비완결성의 근거 |
| SC-3 (속도 상한) | 시간 가변성의 근거 |
| SC-5 (실패 불가제거) | A(x) 축소 → O(x) 축소 |

### 판정

A(x)는 새로운 구조가 아니라, 
SC-5 논증에서 이미 사용된 개념의 명시적 정의이다.

이 보강은 A(x)를 정의하려는 것이 아니라,
A(x)가 오해될 수 있는 모든 경로를 봉인하는 작업이다.

[SC] — SC-5의 보조 정의

## SC-6. 판정의 분산화

**(Necessity of Distributed Judgment)**

### 내용

판정은 단일 중심으로 집중될 수 없다.

### 형식

```
Centralized Judge ⇒ Systemic Fragility
```

### 논증

**경우 A: 정보 부족 유지 (D19 준수)**
- 중앙 판정자 P_c는 I ⊊ Global Info
- 모든 판정을 혼자 → 맥락/지역/예외 누락
- 잘못된 금지/허용 → 옵션 공간 축소

**경우 B: 정보 전역화 시도**
- P_c가 전역 정보 추구 → D19 위반
- 또는 단일 기준 강화 → SC-4(다양성), SC-5(실패 허용)와 충돌

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 완벽히 공정하고 똑똑한 중앙 | "완벽" = 전지성 → D19 위반 |
| 중앙은 원칙만, 실행은 분산 | 이미 분산 판정 = 반례 아님 |
| 정보를 모으되 경계 유지 | 기능상 D19 위반으로 수렴 |

### 판정

판정 구조는 **필연적으로 분산**된다. [필연]

## SC-7. 정체성의 비고정화

**(Non-fixity of Identity)**

### 내용

시스템은 고정된 정체성을 유지할 수 없다.

### 형식

```
Fixed Identity ⇒ |O(x)|↓
```

### 논증

1. 고정 정체성 = 허용/금지 규칙 ID 고정
2. A_ID(x) = A(x) ∩ ID → 행동 공간 절단
3. 환경/사건 변화 → "ID 밖 행동 필요" 확률 증가
4. 금지 고수 → 적응 실패 → 옵션 급감
5. 복원(R)은 종종 r ∉ ID 포함 → 정체성 고정은 복원 제한

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 매우 넓은 정체성 | 행동 제한 없음 = 고정 정체성 아님 |
| 환경 불변 | 사건 누적(비가역)으로 변화 발생, D19로 "불변" 확정 불가 |
| 예외 내장 규칙 | 예외 허용 = 가변 정체성 = 반례 아님 |

### 판정

정체성은 **유지 대상이 아니라 가변 구조**다. [필연]

## SC-8. 결합 귀결 정리

**(Coupling Consequence Theorem)**

### 내용

D10(귀속), D21(완충), D22(비개입·중단), D23(종료·완결)은 독립 필연이지만,
이들이 **결합되지 않을 경우** 시스템은 특정 붕괴 유형 중 하나로 **필연적으로 수렴**한다.
이는 선택적 실패가 아니라 **구조적 귀결 공간**이다.

### 형식

```
¬(D10 ∧ D21 ∧ D22 ∧ D23) ⇒ Collapse ∈ {A, B, C, D}
```

### 전제

- D10(귀속): 선택을 특정 에이전트에 귀속시키는 사상
- D21(완충·자각): 과열 인식 및 옵션 공간 보존 구조
- D22(비개입·중단): 완충의 구조적 구현 (접근 금지선, 외부 중단 권한)
- D23(종료·완결): I_min 보존 상태에서의 구조적 완결

### 붕괴 유형 분류

#### 유형 A — D22 부재 (보호 없는 상태)

**조건**: 전역 불변량 또는 접근 차단 없음

**결과**:
- 단일 국소 사건 → 전역 상태 변경 가능
- 귀속(D10)이 즉시 전역 상태로 전파
- 상태는 정보가 아니라 취약 변수가 됨

**필연 귀결**:
```
¬D22 ⇒ Local Event → Global Propagation → |O(x)|↓
```

**판정**: "구조는 있으나 보호되지 않은 시스템"

---

#### 유형 B — D21 부재 (완충 없는 보호)

**조건**: 전역 불변량은 있으나 동기화/완충 제약 없음

**결과**:
- 전역 상태(클래스)는 유지
- 상대 상태(위상·정렬·정합)는 자유롭게 드리프트
- 국소 성공은 가능하나 전역 기능은 붕괴

**필연 귀결**:
```
¬D21 ⇒ Phase Drift → Cluster Decomposition → Functional Collapse
```

**판정**: "전역 클래스는 유지되나 기능은 붕괴"

#### 유형 C — D10 차단 실패 (숨은 기록 경로)

**조건**: 보호·완충은 있으나 다른 자유도가 기록 매개 역할 수행

**결과**:

- 그러나 간접 기록이 누적
- 장기적으로 상태 오염 발생

**필연 귀결**:

```

**판정**: "겉보기 안정, 내부 침식"

#### 유형 D — 메타 제약 부재 (D20 억제 실패)

**조건**: 초기엔 결합이 성립하나 예외 누적 제어 없음

**결과**:

- 어느 시점에서 구조적 보호 무력화

```

**판정**: "단기 성공, 장기 실패"

### 완전성 논증

위 네 가지 붕괴 유형은:

1. **상호 대체 불가**: 각 유형은 서로 다른 결합 실패에서 발생
2. **상호 환원 불가**: 유형 A의 해결책이 유형 B를 해결하지 못함
3. **완전 집합**: D10·D21·D22·D23 중 하나라도 결합되지 않으면 반드시 이 중 하나로 귀결

본 분류는 D설계도 Core v1.2가 허용하는
운용 레이어의 전체 집합을 기준으로 한 완전성이다.

```
결합 실패 공간 = {¬D22, ¬D21, ¬D10_closure, ¬Meta} 
붕괴 유형 공간 = {A, B, C, D}
사상: 결합 실패 → 붕괴 유형 (전사)
```

주석: ¬Meta는 D20(과열 최적화) 억제 실패(메타 제약 부재)를 의미한다.

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| 다섯 번째 붕괴 유형 존재 | Core v1.2 운용 레이어(D20~D23) + 귀속(D10) 범위 내 완전성 → 새 유형은 새 레이어/새 결합 실패 도입 없이 불가 |

| 부분 결합으로 충분 | 각 유형이 특정 부재에서 발생 → 부분 결합은 해당 유형 붕괴 |

### 판정

**결합 귀결 정리는 SC-1~SC-7의 운용적 완결이다.**
운용 레이어(D20~D23)의 결합 조건이 충족되지 않으면,
시스템은 유형 A~D 중 하나로 **필연적으로** 붕괴한다. [필연]

## SC-9. 완전 서술 불가능성

**(Incompleteness of Description)**

### 내용

어떤 서술 체계도 가능한 상태들의 공간 Ω를 완전하게 포섭할 수 없다.

이는 세계의 '불완전성'에 대한 주장이나 인식론적 한계 선언이 아니라,
선택 구조를 사용하는 모든 서술 체계의 구조적 귀결이다.

### 형식

```
∀ℒ, ∃ω∈Ω s.t. ω ∉ Desc_ℒ(Ω_local)
```

주석: Desc_ℒ는 D19에 의해 항상 Ω_local에 대해서만 정의된다.

즉, 서술의 한계는 표현 능력의 부족이 아니라 접근 가능한 상태 공간의 구조적 제한에서 발생한다.

여기서:
- Ω: 가능한 상태들의 공간 (D0)
- Ω_local: 국소적으로 접근 가능한 상태 공간 (D19)
- ℒ: 서술 체계 (구분 + 선택을 통한 형식적 절차)
- Desc_ℒ(Ω_local): ℒ이 구분·선택을 통해 포섭한 상태들의 집합

### 논증

1. D0 (존재)에 의해, Ω ≠ ∅
2. D1 (구분)에 의해, Ω는 구분 가능한 부분들로 분해된다
3. D9 (선택)에 의해, 구분된 상태들 중 일부만이 고정된다
4. D13 (옵션 공간 축소)에 의해, 선택은 Ω의 부분집합만을 비가역적으로 제거한다
5. 이 제거는 항상 부분적이며, Ω 전체를 소거하지 않는다
6. D19 (폐쇄 경계)에 의해, 서술 체계 ℒ는 Ω_local에만 접근 가능하다
7. 따라서 어떤 서술 체계 ℒ도 Ω 전체를 포섭할 수 없다

### 반례 검증

| 반례 시도 | 판정 |
|----------|------|
| "무한한 서술 능력을 가진 체계" | D19 (폐쇄 경계) 위반 — 전역 접근 불가 |
| "Ω 자체를 서술 대상에서 제외" | D0 위반 — 서술 대상이 없으면 서술 자체 불가 |
| "선택 없이 구분만으로 서술" | D9 필연 — 구분만으로는 고정 불가, 선택 필수 |
| "모든 상태를 동시에 고정" | D13 위반 — 선택은 축소를 동반, 동시 전체 고정 불가 |
| "확률 분포로 모든 상태를 서술" | D9 위반 — 확률은 선택 이전의 가능성 구조이며, 서술은 선택(D9) 이후의 고정 결과에만 적용됨 |

### 판정

완전 서술은 목표가 아니라 구조적으로 배제된 상태다. [필연]

### Core 연결

- D0 (존재): Ω ≠ ∅
- D1 (구분): 분할 가능성
- D9 (선택): 부분 고정
- D13 (옵션 공간 축소): 비가역 축소
- D19 (폐쇄 경계): 전역 접근 불가

### SC-1~SC-8과의 관계

SC-9는 SC-1~SC-8의 결과가 아니라,
그들이 작동하기 위해 항상 배경으로 성립하는 메타 귀결이다.

- SC-2 (전지적 최적화 불가)와 동형: 전역 정보 접근 불가 → 완전 서술 불가
- SC-6 (판정 분산화)의 근거: 완전 서술 불가 → 단일 판정 불가

### 귀결

- 서술은 항상 Ω의 부분 집합에 대해서만 성립한다
- 잔여는 오류나 결함이 아니라, 구조가 지속되기 위해 필연적으로 남는 결과다
- D설계도 자체도 이 귀결의 적용을 받는다 (OC-8 자기완결 금지와 정합)

## SC-9.1 코딩-정정-게이지 귀결 체인

**(Coding-Correction-Gauge Consequence Chain)**

### 내용

SC-9(완전 서술 불가능성)로부터 다음 연쇄가 구조적으로 따라온다:

```
정보 상한 → 코딩 필연 → 오류정정 필연 → 게이지 필연
```

### 논증

1. **정보 상한 → 코딩 필연** (D19 + SC-9)
   - 경계(D19)가 저장 가능한 정보량에 상한을 부과한다
   - 상태를 그대로 보존할 수 없으므로, 코딩된 부분공간만 안정적으로 유지된다
   - 따라서 코딩은 선택이 아니라 구조적 필연이다

2. **코딩 → 오류정정 필연** (D12 + D15)
   - 코딩된 정보를 안정적으로 유지하려면 국소 손실을 허용해야 한다
   - 국소 손실 허용 + 전역 일관성 요구 → 오류정정 구조 필연
   - 이는 D12(안정성)의 정보 이론적 표현이다

3. **오류정정 → 게이지 필연** (P-3 대응)
   - 오류정정이 가능하려면 동일 물리 상태를 여러 표현으로 나타내는 중복성이 필요하다
   - 이 중복성이 게이지 구조다
   - 게이지 불변성 + 국소성 → 곡률 제곱형 동역학 강제 (Physics P-3과 정합)

### 판정

코딩-정정-게이지는 SC-9의 물리적 귀결이다. [SC]

### Core 연결

| 항목 | 연결 |
|------|------|
| D19 | 정보 상한의 근거 |
| D12 | 안정성 → 오류정정 |
| SC-9 | 완전 서술 불가 → 코딩 필연 |
| P-3 | 게이지 → 장 구조 |
